# ADR 0001: 圖構建方式（Geographic k-Nearest Neighbors）

## 背景 (Context)
YouBike 站點分佈密集且具有地理關聯，模型需要了解鄰近站點間的流量互動。資料來源僅提供經緯度與即時狀態，因此需自行構建圖結構供圖神經網路使用。

## 決策 (Decision)
- 使用站點經緯度計算 pairwise 地理距離 (Haversine 或近似平面距離)。
- 對每個節點連接固定的 k 個最近鄰，預設 k=8，可依模型需求調整 (5-12)。
- 邊權重採用距離的高斯核 `exp(-d / σ)`，σ 以 0.01 度作為初始值，可調整以控制權重衰減。
- 生成的圖以 JSON 儲存 nodes/edges，同步輸出鄰接矩陣供模型載入。

## 理由 (Rationale)
- kNN 可控制邊數，避免完全圖造成計算成本過高。
- 地理距離是最容易取得且直觀的相似性指標，適用於站點密集的都市環境。
- 固定 k 有助於保持圖的稀疏性，利於批次訓練與推論。

## 替代方案 (Alternatives)
1. **固定半徑**：設定距離閾值 (例如 500 公尺) 建立連結，但在站點密度不均時會造成孤立節點或過多邊。
2. **路網距離**：使用實際騎乘時間或路網距離，準確度更高，但需要額外資料與計算資源。
3. **社群偵測/分群圖**：透過歷史流量建立社群圖，適合長期趨勢，但需要充分的歷史資料與複雜的更新流程。

## 後果 (Consequences)
- **正面**：構建簡單、易於實作；對大部分市區站點能捕捉鄰近互動。
- **負面**：偏遠站點可能因距離過遠僅能連接較弱的鄰居，需配合虛擬節點或加強特徵。
- **後續工作**：
  - 分析不同 k 值對模型效果的影響。
  - 觀察尖峰/離峰時段的邊權重是否需自適應調整。
  - 若取得歷史騎乘 OD 資料，可評估改為基於流量的加權圖。
